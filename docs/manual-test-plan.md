## 실행 환경

### 필수 하드웨어

- Raspberry Pi (터치스크린 연결)
- USB 웹캠
- 스피커 (3.5mm 또는 HDMI)
- (선택) USB 마이크

### 실행 명령

```bash
cd ~/RIO
venv/bin/python scripts/live_interaction_test.py
```

- `--real-services`: 스마트홈/날씨 실제 HTTP 요청 모드
- `--debug`: 웹캠 인셋 + 상태 사이드바 (CPU 부하 높음)

### 테스트 보조 명령어 (시뮬레이션)

하드웨어로 재현이 어려운 경우에만 보조적으로 사용합니다.

| 명령 | 용도 |
|------|------|
| `/tap` | 터치 탭 시뮬레이션 |
| `/stroke` | 쓰다듬기 시뮬레이션 |
| `/timer [label]` | 타이머 만료 즉시 발생 |
| `/gesture [name]` | 제스처 시뮬레이션 (wave, finger_gun, v_sign, head_left, head_right, peekaboo) |
| `/face [pos]` | 얼굴 위치 시뮬레이션 (left, center, right, lost) |
| `/status` | 현재 상태 출력 |

### 판정 기준

- **PASS**: 기대 동작이 모두 관찰됨 (화면 표정 + 사운드 + 상태 전이)
- **FAIL**: 기대 동작과 다르거나 오류 발생
- **SKIP**: 하드웨어 미지원 등으로 테스트 불가

### 확인 포인트

모든 테스트에서 아래 항목을 관찰합니다:
- **화면**: 표정이 바뀌는가, UI가 전환되는가
- **사운드**: 효과음이 재생되는가
- **HUD**: 텍스트가 표시되는가
- **상태**: `/status`로 Context/Activity/oneshot이 기대대로인가

## 담당 배분

MVP 테스트를 두 명이 나눠서 수행합니다.

| 담당 | 테마 | 섹션 | 항목수 |
|------|------|------|--------|
| **Tester A** | 존재감 / 감정 / 반응 | SYS (12) + INT (11) + ONE (5) + SCN (5) | **33** |
| **Tester B** | 명령 / 정책 / 운영 | VOICE (18) + POL (8) + OPS (5) | **31** |

- Tester A는 **"RIO가 사람을 인식하고 감정적으로 반응하는가"** 를 검증합니다.
- Tester B는 **"RIO가 명령을 올바르게 처리하고 예외 상황에서 안정적인가"** 를 검증합니다.
- Phase 2 테스트(10개)는 MVP 완료 후 별도 수행합니다.
