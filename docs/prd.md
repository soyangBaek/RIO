# [Project] 데스크탑 반려 로봇 "RIO" 통합 문서

## 1. 제품 정의 (Product Definition)
* **제품명:** RIO (Interactive Smart-Home Companion)
* **컨셉:** BMO의 감성과 EMO의 지능을 결합한 데스크탑 반려 로봇.
* **핵심 역할:** LG ThinQ와 연동되어 집안 가전을 제어하는 '살아있는' 스마트홈 허브.

## 2. 핵심 목표
* **감성 교감:** 비전 인식 및 터치 센서를 통한 물리적/정서적 상호작용.
* **스마트 제어:** 음성 명령을 통한 ThinQ 가전 제어 및 상태 피드백 제공.
* **엔터테인먼트:** 미니 게임(핑퐁, 갤로그) 및 사용자 분석 기반 댄스 모드.

## 3. 기술 스택 (Technical Stack)
* **Main OS:** Raspberry Pi 또는 Jetson Nano 기반 Linux
* **Vision:** OpenCV, MediaPipe (Face/Gesture/Eye Tracking)
* **Voice:** STT (Google/OpenAI Whisper), TTS
* **Network:** HTTP/JSON (ThinQ Server 연동)
* **UI:** Lottie (애니메이션), React 또는 Python Tkinter/PyQt

---

## 4. 상세 기능 명세

### 4.1 ThinQ 연동 및 통신
RIO는 사용자의 음성을 해석하여 로컬 홈 클라이언트로 명령을 전달함.
* **통신 방식:** HTTP / Method: `PUT`
* **Endpoint:** `http://[Home-Client-IP]/device/control`
* **Request Body:**
    ```json
    {
      "content": "에어컨 켜줘"
    }
    ```
* **제어 대상:** 에어컨, 조명, 로봇 청소기, TV, 스피커 등.

### 4.2 입력 수단별 로직
| 구분 | 입력 상황 | 동작 로직 |
| :--- | :--- | :--- |
| **Voice** | "RIO dance!" | 음악 분석 후 비트에 맞춰 댄스 모드 활성화 |
| | "사진 찍어줘" | 5초 카운트다운 후 카메라 캡처 및 저장 |
| | "~분 뒤 알려줘" | 내부 타이머 설정 및 종료 시 알람 |
| **Vision** | 시선 추적 | 얼굴 좌표($$x, y$$)에 맞춰 모터 및 눈동자 이동 |
| | 빵야! (Gesture) | $$L$$자 손가락 인식 시 뒤로 쓰러지는 모션 |
| | 숨바꼭질 | 얼굴 가려짐 인식 후 재등장 시 환희 모션 |
| **Physical** | 터치 센서 | 머리/옆면 터치 시 가르릉거리며 하트 노출 |

## 5. UI Layer 구조 및 가이드
RIO의 화면은 총 3개의 레이어로 구성되어 독립적으로 렌더링됨.

* **Layer 1 (Core Face):** 기초 감정 및 눈동자 움직임 (Lottie 기반).
* **Layer 2 (Action Overlay):** 상황별 오브젝트 (꽃가루, 뿅망치, 아이콘).
* **Layer 3 (System HUD):** 텍스트 정보 (STT 결과, 타이머 숫자, 알림창).

## 7. 구현 주의사항 (Developer Note)
1.  **Transition:** 모드 전환 시 눈동자의 위치가 끊기지 않도록 Tweening 라이브러리를 사용하여 부드럽게 처리할 것.
2.  **Dimming:** 자장가나 수면 모드 시 화면의 전체 Brightness를 `lerp` 함수 등을 이용해 서서히 낮출 것.
3.  **Exception:** ThinQ 제어 실패 시 단순 텍스트 표기보다는 레이어 1의 표정(> <)을 적극 활용하여 로봇의 '미안함'을 표현할 것.
