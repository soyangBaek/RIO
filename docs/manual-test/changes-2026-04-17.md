## 4. MT-VOICE-06: 타이머 등록 및 알림

### 수정 내용
- 파일: `scripts/live_interaction_test.py`, `src/app/adapters/display/hud.py`, `src/app/domains/behavior/effect_planner.py`, `src/app/adapters/speaker/sfx.py`, `src/app/main.py`
- **메인 루프에 scheduler 폴링 추가**: `for due in rio.scheduler.poll_due(): rio.process_event(due)` — 이게 없으면 타이머 만료 이벤트가 발생하지 않음 (치명적 버그 수정)
- **등록 확인 HUD**: `"Timer set: 5 min"` 형태 표시
- **등록 확인 SFX**: `timer_registered` → `timer_ring.mp3`
- **만료 알림 SFX**: `alert` → `timer_ring.mp3` (1회 재생)
- **자동 해제 15초**: `ALERT_AUTO_DISMISS_SECONDS = 15.0` — Alerting 진입 시 threading.Timer 스케줄 → 15초 후 `SYSTEM_ALERT_TIMEOUT` 자동 발행 → Idle 복귀
- **수동 해제**: "확인" 입력 시 즉시 Idle 전이 + 자동 해제 타이머 취소

### 테스트 절차

**단기 테스트 (빠른 확인)**
1. `10초 뒤에 알려줘` 입력
2. "Timer set: 10s" HUD + timer_ring.mp3 확인
3. 10초 대기 → 알림음 + AlertUI 확인
4. **확인 입력** → Idle 복귀, 알림음/UI 사라짐 확인

**장기 테스트 (MT-VOICE-06 원본)**
1. `5분 뒤에 알려줘` 입력
2. "Timer set: 5 min" HUD 확인
3. 5분 대기 → 알림
4. `확인` 또는 방치하여 15초 후 자동 해제 확인

**자동 해제 테스트**
1. `5초 뒤에 알려줘` 입력
2. 5초 후 알림 울림
3. **아무것도 입력하지 말고** 15초 대기
4. 자동으로 Idle 복귀 확인

### 기대 동작
- 등록 직후 `Idle` 복귀 (Listening 에 머물지 않음)
- 알림음은 1회만 재생 (1.75초)
- "확인" 입력 시 즉시 해제
- 미입력 시 15초 후 자동 해제

### 관련 테스트 케이스
- MT-VOICE-06, MT-VOICE-18 (확인 해제)
- MT-INT-10 (타이머 만료 알림)
- MT-SCN-01, MT-SCN-02 (졸음/Away 중 알림 우선)
- MT-POL-08 (알림 해제 후 복귀)

## 6. MT-POL-07: 댄스 중 "취소"로 조기 종료

### 수정 내용
- 파일: `src/app/core/state/activity_fsm.py`, `src/app/main.py`
- **FSM 전이 추가**: `EXECUTING` + `VOICE_INTENT_DETECTED(system.cancel 또는 system.ack)` + `active_kind ∈ {DANCE, GAME}` → `IDLE`
- **정리 로직**: `_handle_long_action_cancel()` — 취소로 인한 DANCE 종료 시:
  - `sfx.stop("dance")` — 음악 즉시 정지
  - `_dance_timer.cancel()` — 지연 `TASK_SUCCEEDED` 발행 방지
- 게임 모드는 `extended_state`에서 `system.cancel` 발생 시 `ui_mode=None` 으로 이미 해제됨

### 테스트 절차
**댄스 취소**
1. `춤춰` 입력
2. 1~5초 사이 `취소` 입력
3. 즉시 음악 정지 및 Idle 복귀 확인
4. `/status`로 `Activity: Idle`, `kind: None` 확인

**즉시 취소 (엣지 케이스)**
1. `춤춰` + Enter
2. **바로** `취소` + Enter (딜레이 없이)
3. 음악이 들리지 않고 즉시 Idle 복귀 확인

**게임 모드 취소**
1. `게임 모드로 바꿔줘` 입력 → `ui_mode: game`
2. `취소` 입력 → `ui_mode: None`

### 기대 동작
- 음악 즉시 정지
- 얼굴이 dance_face → 기본 표정 복귀
- `activity.state.changed` 이벤트 발행 (recent 로그 확인)

## 7. 공통 UX 개선: 터미널 입력 시 startled 제거

### 수정 내용
- 파일: `src/app/core/state/reducers.py`
- `VOICE_ACTIVITY_STARTED + !face_present` → startled 발동 조건에 `event.source != "audio.terminal_input"` 추가
- 터미널로 타이핑하는 경우(사용자가 키보드를 보고 있어 얼굴 미탐지) startled 오발동 방지
- 실제 마이크 음성(`audio.vad` 등)은 기존대로 startled 유지 (MT-SYS-04 의도 보존)

### 테스트 절차
1. 얼굴이 카메라에 보이지 않는 상태에서 터미널에 아무 명령 입력
2. startled 효과음/표정이 나오지 않는지 확인

### 기대 동작
- 터미널 입력 시 `startled` sfx 재생되지 않음
- 모든 명령(`춤춰`, `사진`, `에어컨 켜줘`, `5분 뒤에 알려줘` 등)에서 깨끗한 진입
